## 프로젝트 본질

### 한 줄 요약

**"한국어 표준 용어 기반의 용어 표준화와 데이터베이스 설계 관리를 통합적으로 수행하는 시스템"**

### 핵심 목적

이 프로젝트는 **데이터 표준화**, **용어 일관성**, 그리고 **체계적인 데이터베이스 설계 관리**를 보장하기 위한 도구입니다.

#### 해결하는 문제

1. **용어 불일치 문제**
   - 같은 개념을 다른 이름으로 표현하는 문제
   - 예: "사용자" vs "유저" vs "회원"
   - 예: "USER" vs "MEMBER" vs "ACCOUNT"

2. **표준 용어 관리의 어려움**
   - 표준 용어집을 수동으로 관리하는 번거로움
   - 용어 간 매핑 관계 추적의 어려움
   - 도메인 표준과의 동기화 문제

3. **데이터베이스 설계 시 용어 생성의 어려움**
   - 한국어 용어를 영문 컬럼명으로 변환하는 과정의 복잡성
   - 여러 가능한 조합 중 적절한 선택의 어려움
   - 표준 용어집과의 일치 여부 확인의 어려움

4. **산재된 설계 정보**: 데이터베이스, 테이블, 컬럼 등의 설계 정보가 파편화되어 관리되는 문제

### 타겟 사용자

- **데이터 아키텍트**: 데이터 표준화, 용어 관리, 데이터 모델 관리
- **데이터베이스 설계자**: 표준 용어 기반 컬럼명/도메인명 생성 및 설계서 작성
- **개발자**: 일관된 용어 사용을 위한 참조 도구 및 설계 참조
- **데이터 관리자**: 용어집, 도메인 표준 및 데이터베이스 설계 자산 관리

## 핵심 비즈니스 도메인

이 프로젝트는 기능적으로 **용어 표준화(Term Standardization)**와 **데이터베이스 설계(Database Design)** 두 개의 독립적인 모듈로 구성됩니다.

### 용어 표준화 (Term Standardization)

#### 1. Vocabulary (단어집)

**정의**: 한국어 표준 용어와 그에 대응하는 영문 약어/전체명을 관리하는 사전

**비즈니스 가치**: 표준 용어의 중앙 집중식 관리, 용어 일관성 보장, 용어 재사용 촉진

#### 2. Domain (도메인)

**정의**: 공통표준도메인 정보를 관리하는 표준 도메인 사전

**비즈니스 가치**: 도메인 표준의 중앙 관리, 단어집과 도메인 간 자동 동기화, 데이터 타입 표준화

#### 3. Term (용어)

**정의**: 실제 데이터베이스에서 사용할 용어명, 컬럼명, 도메인명을 조합한 최종 용어

**비즈니스 가치**: 표준 용어 기반 데이터베이스 설계, 용어 일관성 자동 검증, 매핑 상태 추적

### 데이터베이스 설계 (Database Design)

데이터베이스의 논리적/물리적 설계를 구성하는 요소를 관리합니다.

- **Database**: 관리 대상 데이터베이스 인스턴스 정보
- **Entity**: 데이터베이스의 논리적 데이터 단위
- **Attribute**: 엔터티에 포함되는 속성 정보
- **Table**: 물리 데이터베이스에 생성되는 테이블 명세
- **Column**: 테이블을 구성하는 컬럼의 상세 정보

## 데이터 모델과 관계

현재 아키텍처에서 **용어 표준화** 모듈과 **데이터베이스 설계** 모듈은 **서로 독립적**입니다.

### 1. 용어 표준화 모듈 내 관계

```
┌─────────────────┐
│  Vocabulary     │ (단어집)
│  - standardName │
│  - abbreviation │
│  - domainCategory│
└────────┬────────┘
         │
         │ domainCategory 매핑
         │
┌────────▼────────┐
│    Domain       │ (도메인)
│  - domainGroup  │
│  - domainCategory│
│  - standardDomainName│
└─────────────────┘
         │
         │
         │ 참조
         │
┌────────▼────────┐
│     Term        │ (용어)
│  - termName     │ ← Vocabulary.standardName 참조
│  - columnName   │ ← Vocabulary.abbreviation 참조
│  - domainName   │ ← Domain.standardDomainName 참조
└─────────────────┘
```

- **Vocabulary ↔ Domain**: `domainCategory` 필드를 통해 N:M 관계를 맺으며, 동기화 API를 통해 `Vocabulary`의 `domainGroup`이 자동으로 설정됩니다.
- **Vocabulary/Domain → Term**: `Term`은 `Vocabulary`의 단어들과 `Domain`의 표준 도메인명을 참조하여 생성되며, 매핑 검증 로직을 통해 일관성을 유지합니다.

### 2. 데이터베이스 설계 모듈 내 관계

데이터베이스 설계 모듈 내의 엔터티(Database, Entity, Table 등)들은 현재 시스템 내에서 강제된 참조 관계없이 독립적으로 관리됩니다. 각 항목은 UI를 통해 사람이 인지하는 논리적 관계를 가집니다.

## 주요 사용 사례

### Use Case 1: 표준 용어 등록 및 관리

**시나리오**:

1. 데이터 아키텍트가 새로운 표준 용어를 등록
2. 한국어 표준명, 영문약어, 영문명 입력
3. 도메인 분류명 지정
4. 금지어 및 이음동의어 설정

**결과**:

- 표준 용어집에 용어 추가
- 도메인과 자동 매핑
- 다른 용어 생성 시 재사용 가능

### Use Case 2: 용어 자동 생성 (Term Generator)

**시나리오**:

1. 개발자가 한국어 용어 입력 (예: "사용자 이름")
2. 시스템이 단어집을 기반으로 가능한 조합 생성
   - "사용자" → ["USER", "MEMBER", ...]
   - "이름" → ["NAME", "FIRST_NAME", ...]
3. 모든 조합 생성 (Cartesian Product)
   - "USER_NAME", "USER_FIRST_NAME", "MEMBER_NAME", ...
4. 개발자가 적절한 조합 선택
5. 도메인명 매핑 확인

**결과**:

- 표준 용어 기반 컬럼명 생성
- 여러 가능한 조합 제시
- 매핑 상태 자동 검증

### Use Case 3: 용어 검증 및 자동 수정

**시나리오**:

1. 용어 업로드 또는 생성 시 매핑 검증
2. 매핑 실패 시 원인 분석
   - 단어집에 없는 단어 발견
   - 도메인명 불일치
3. 자동 수정 제안
   - 누락된 단어를 단어집에 추가 제안
   - 유사한 단어 제안
   - 도메인명 수정 제안

**결과**:

- 용어 품질 자동 검증
- 표준 준수 자동화
- 수정 제안으로 생산성 향상

### Use Case 4: 대량 용어 업로드 및 동기화

**시나리오**:

1. Excel 파일로 대량 용어 업로드
2. 시스템이 자동으로 매핑 검증
3. 실패한 용어 목록 표시
4. 자동 수정 제안 및 일괄 적용
5. 도메인 동기화 수행

**결과**:

- 대량 용어 일괄 처리
- 품질 검증 자동화
- 표준 준수 보장

## 아키텍처 특징

### 1. 파일 기반 저장소

**특징**:

- 전통적인 데이터베이스 대신 JSON 파일 사용
- `static/data/` 디렉토리 아래에 `vocabulary`, `domain`, `database` 등 기능별 하위 디렉토리를 두어 데이터 관리
- 파일별로 독립적인 데이터 관리

**장점**:

- 간단한 배포 (파일 복사만으로 백업/복원)
- 버전 관리 용이 (Git으로 추적 가능)
- 데이터베이스 설정 불필요

**단점**:

- 동시성 제어 필요 (파일 락 메커니즘 구현됨)
- 대용량 데이터 처리 제한
- 트랜잭션 없음 (원자적 쓰기 패턴으로 보완)

### 2. 다중 파일 지원

**특징**:

- 모든 엔터티는 여러 개의 JSON 파일을 지원
- 파일별 독립적인 데이터 관리
- 파일 간 매핑 관계 설정

**비즈니스 의미**:

- 프로젝트별/부서별 용어집 분리 관리
- 표준 용어집과 프로젝트 용어집 구분

### 3. 매핑 기반 검증 시스템 (용어 표준화 모듈 한정)

**특징**:

- Vocabulary ↔ Domain: `domainCategory` 기반 자동 매핑
- Vocabulary → Term: 언더스코어 분리 후 각 단어 검증
- Domain → Term: 정확 일치 검증

**비즈니스 의미**:

- 표준 준수 자동화
- 비표준 용어 사용 방지
- 데이터 품질 보장

### 4. 모듈 간 분리 (Decoupling)

**특징**:

- **"용어 표준화"** 모듈과 **"데이터베이스 설계"** 모듈은 아키텍처상 **완전히 분리**되어 있습니다.
- 두 모듈 간 데이터 직접 참조나 기능적 종속성이 없습니다.

**비즈니스 의미**:

- 각 기능 모듈의 독립적인 유지보수 및 확장이 용이합니다.
- 향후 두 모듈을 연계하는 새로운 기능(예: 설계 시 표준 용어 자동 추천)을 구현할 수 있는 유연성을 제공합니다.

## 프로젝트의 독특한 점

### 1. 한국어 중심의 용어 관리

대부분의 용어 관리 시스템이 영문 중심인데, 이 프로젝트는 **한국어 표준 용어를 중심**으로 설계되었습니다.

- 한국어 → 영문 변환 자동화
- 한국어 표준 용어 우선 관리
- 한국어 기반 검색 및 필터링

### 2. 매핑 기반 검증 시스템

단순한 용어 사전이 아니라, **용어 간 매핑 관계를 추적하고 검증**하는 시스템입니다.

- Vocabulary ↔ Domain 자동 매핑
- Term → Vocabulary/Domain 매핑 검증
- 매핑 실패 시 자동 수정 제안

### 3. 파일 기반의 단순함

복잡한 데이터베이스 없이 **JSON 파일만으로 모든 데이터 관리**합니다.

- 배포 간단 (파일 복사만)
- 버전 관리 용이 (Git)
- 데이터베이스 설정 불필요

### 4. 실용적인 자동화

이론적인 표준화가 아니라, **실제 개발 과정에서 사용할 수 있는 자동화**를 제공합니다.

- 용어 자동 생성 (Term Generator)
- 매핑 자동 검증
- 자동 수정 제안

## 결론

DbManager는 단순한 용어 관리 시스템이 아니라, **데이터 표준화와 용어 일관성을 보장하는 종합적인 플랫폼**입니다.

### 핵심 가치 제안

1. **표준 용어의 중앙 관리**: Vocabulary로 표준 용어 집중 관리
2. **자동화된 용어 생성**: Term Generator로 표준 기반 용어 자동 생성
3. **자동 검증 시스템**: 매핑 기반으로 표준 준수 자동 검증
4. **실용적인 도구**: 실제 개발 과정에서 바로 사용 가능

### 프로젝트의 정체성

**"데이터 표준화를 위한 실용적인 용어 관리 플랫폼"**

- 이론이 아닌 실전 도구
- 복잡함이 아닌 단순함
- 수동이 아닌 자동화
- 개별이 아닌 통합 관리
