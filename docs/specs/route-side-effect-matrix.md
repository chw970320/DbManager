# Route Side-Effect Matrix

This matrix records side effects for the upload, sync, and validation routes touched by the 2026-06 upload/sync cleanup.

| Route or helper                          | Load path                                                                               | Cached read path | Save path                                                            | Invalidation required                                             | Generator cache impact | Preview/apply behavior                                                                             |
| ---------------------------------------- | --------------------------------------------------------------------------------------- | ---------------- | -------------------------------------------------------------------- | ----------------------------------------------------------------- | ---------------------- | -------------------------------------------------------------------------------------------------- |
| `GET /api/entity/upload`                 | `loadEntityData(filename)`                                                              | None             | None                                                                 | No                                                                | None                   | Read-only upload metadata                                                                          |
| `POST /api/entity/upload`                | XLSX parser input only                                                                  | None             | `mergeEntityData(parsedEntries, replaceExisting, filename)`          | Not in route; registry persistence owns file update               | None                   | Always replace/merge upload, post-process mode currently `none`                                    |
| `GET /api/attribute/upload`              | `loadAttributeData(filename)`                                                           | None             | None                                                                 | No                                                                | None                   | Read-only upload metadata                                                                          |
| `POST /api/attribute/upload`             | XLSX parser input only                                                                  | None             | `mergeAttributeData(parsedEntries, replaceExisting, filename)`       | Not in route; registry persistence owns file update               | None                   | Always replace/merge upload, post-process mode currently `none`                                    |
| `GET /api/table/upload`                  | `loadTableData(filename)`                                                               | None             | None                                                                 | No                                                                | None                   | Read-only upload metadata                                                                          |
| `POST /api/table/upload`                 | XLSX parser input only                                                                  | None             | `mergeTableData(parsedEntries, replaceExisting, filename)`           | Not in route; registry persistence owns file update               | None                   | Always replace/merge upload, post-process mode currently `none`                                    |
| `GET /api/column/upload`                 | `loadColumnData(filename)`                                                              | None             | None                                                                 | No                                                                | None                   | Read-only upload metadata                                                                          |
| `POST /api/column/upload`                | XLSX parser input only                                                                  | None             | `mergeColumnData(parsedEntries, replaceExisting, filename)`          | Not in route; registry persistence owns file update               | None                   | Always replace/merge upload, post-process mode currently `none`                                    |
| `POST /api/term/sync`                    | `loadData('term')`, `loadData('vocabulary')`, `loadData('domain')`                      | None             | `saveData('term', finalData, filename)` when apply mode runs         | Yes: `invalidateDataCache('term', filename)` after save succeeds  | None                   | `apply=false` is read-only preview; apply saves and invalidates                                    |
| `GET /api/column/sync-term`              | `loadData('column')`, `loadData('term')`, `loadData('domain')`                          | None             | None                                                                 | No                                                                | None                   | Read-only status                                                                                   |
| `POST /api/column/sync-term`             | `loadData('column')`, `loadData('term')`, `loadData('domain')`                          | None             | `saveData('column', finalData, colFile)` when `apply && updated > 0` | Yes: `invalidateDataCache('column', colFile)` after save succeeds | None                   | `apply=false` is read-only preview; apply saves only when changes exist                            |
| `runUploadPostProcess` relation step     | Delegates to route fetch                                                                | None             | None directly                                                        | Delegated to called routes                                        | None                   | `validate-only` calls canonical validation; `validate-sync` also calls alignment sync              |
| `POST /api/alignment/sync` relation step | Delegates to vocabulary, term, canonical relation validation, column, and report routes | None directly    | None directly                                                        | Delegated to called sync routes                                   | None                   | `apply` propagates to vocabulary/term/column sync; relation step is canonical read-only validation |
| `GET /api/validation/design-relations`   | `runDesignRelationValidation(...)`                                                      | None in route    | None                                                                 | No                                                                | None                   | Canonical read-only relation validation                                                            |

Notes:

- Upload routes intentionally expose required-column metadata but do not perform cache invalidation in route code.
- `term/sync` and `column/sync-term` invalidate only after a successful save so preview and failed-save paths cannot evict warm data.
- Non-ERD relation diagnostics should call `GET /api/validation/design-relations`; `GET /api/erd/relations` remains compatibility-only.
