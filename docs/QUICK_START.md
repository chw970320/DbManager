# 🚀 빠른 시작 가이드

DbManager 프로젝트를 빠르게 시작하는 가이드입니다.

**대상**: 모든 사용자 (데이터 아키텍트, DB 설계자, 개발자)

---

## 📋 전제 조건

- [Node.js](https://nodejs.org/) (v18 이상 권장)
- [pnpm](https://pnpm.io/) (패키지 매니저)
- [Docker](https://www.docker.com/) (선택 사항)

---

## ⚡ 빠른 설치 및 실행

### 1. 저장소 복제

```bash
git clone https://github.com/chw970320/DbManager.git
cd DbManager
```

### 2. 의존성 설치

```bash
pnpm install
```

### 3. 개발 서버 실행

```bash
pnpm run dev
```

### 4. 브라우저 접속

브라우저에서 `http://localhost:5173`으로 접속하세요.

---

## 🎯 첫 번째 용어 생성하기

### 단계 1: 단어집에 표준 용어 등록

1. 홈페이지에서 **"단어집"** 메뉴 클릭 (`/browse`)
2. **"새 단어 추가"** 버튼 클릭
3. 다음 정보 입력:
   - **표준단어명**: "사용자" (한국어)
   - **영문약어**: "USER" (컬럼명에 사용)
   - **영문명**: "User" (전체 영문명)
   - **설명**: "시스템 사용자"
   - **도메인 분류명**: "사용자분류" (선택 사항)
4. **저장** 버튼 클릭

### 단계 2: 도메인 동기화 (선택 사항)

1. **"도메인"** 메뉴로 이동 (`/domain/browse`)
2. 도메인 정보가 있다면, 단어집과 도메인을 동기화:
   - 단어집 페이지에서 **"도메인 동기화"** 버튼 클릭
   - `domainCategory`가 일치하는 도메인과 자동 매핑됩니다

### 단계 3: 용어 생성

1. **"용어"** 메뉴로 이동 (`/term/browse`)
2. **"용어 변환기"** 섹션에서:
   - 한국어 용어 입력: "사용자 이름"
   - 시스템이 자동으로 가능한 조합 생성:
     - "USER_NAME", "USER_FIRST_NAME", "MEMBER_NAME" 등
3. 적절한 조합 선택
4. 도메인명 선택 (자동 추천 가능)
5. **"용어 추가"** 버튼 클릭

### 단계 4: 매핑 검증 확인

생성된 용어의 매핑 상태를 확인:

- ✅ **용어명 매핑**: 단어집의 `standardName`과 일치하는지 확인
- ✅ **컬럼명 매핑**: 단어집의 `abbreviation`과 일치하는지 확인
- ✅ **도메인 매핑**: 도메인의 `standardDomainName`과 일치하는지 확인

## 📚 주요 기능 빠른 둘러보기

### 1. 단어집 관리 (`/browse`)

- **조회**: 페이지네이션, 정렬, 필터링 지원
- **추가/수정/삭제**: 웹 인터페이스에서 직접 관리
- **파일 관리**: 여러 단어집 파일 생성 및 관리
- **업로드**: Excel 파일로 대량 등록
- **다운로드**: Excel 파일로 내보내기

### 2. 도메인 관리 (`/domain/browse`)

- **조회**: 도메인 정보 검색 및 필터링
- **추가/수정/삭제**: 도메인 정보 관리
- **업로드**: Excel 파일로 대량 등록
- **동기화**: 단어집과 자동 매핑

### 3. 용어 관리 (`/term/browse`)

- **조회**: 용어 검색 및 필터링
- **자동 생성**: 용어 변환기로 표준 기반 용어 생성
- **매핑 검증**: 단어집/도메인과의 매핑 상태 확인
- **자동 수정**: 매핑 실패 시 수정 제안 및 자동 적용
- **업로드/다운로드**: Excel 파일 지원

## ❓ 문제 해결

### 포트가 이미 사용 중인 경우

```bash
# 다른 포트로 실행
pnpm run dev -- --port 3000
```

### 의존성 설치 오류

```bash
# 캐시 정리 후 재설치
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### Docker 실행 오류

```bash
# 컨테이너 및 이미지 정리
docker-compose down
docker system prune -a
docker-compose up --build
```
