## 화면 이동

- 상단 메뉴는 `표준 용어`, `DB 설계`, `운영 · 품질`의 3개 그룹으로 구성됩니다.
- 각 화면 상단 breadcrumb에서 현재 메뉴 그룹과 현재 페이지를 함께 확인할 수 있습니다.
- `lv1` breadcrumb를 클릭하면 다른 `lv1` 그룹(`표준 용어`, `DB 설계`, `운영 · 품질`)으로 이동할 수 있습니다.
- `lv2` breadcrumb를 클릭하면 현재 `lv1` 그룹 안의 다른 `lv2` 메뉴로 바로 이동할 수 있습니다.
- breadcrumb는 데스크탑/태블릿 폭에서만 표시되며, 작은 화면에서는 숨겨집니다.

## 단어집 관리

### 단어집이란?

**단어집(Vocabulary)**은 한국어 표준 용어와 그에 대응하는 영문 약어/전체명을 관리하는 사전입니다.

**핵심 개념**:

- **표준단어명** (`standardName`): 한국어 표준 용어 (예: "사용자")
- **영문약어** (`abbreviation`): 데이터베이스 컬럼명에 사용 (예: "USER")
- **영문명** (`englishName`): 전체 영문명 (예: "User")

### 단어 추가하기

1. **단어집 페이지**로 이동 (`/browse`)
2. **"새 단어 추가"** 버튼 클릭
3. 필수 정보 입력:
   - 표준단어명 (한국어)
   - 영문약어
   - 영문명
   - 설명
4. 선택 정보 입력:
   - 도메인 분류명 (`domainCategory`)
   - 형식단어 여부 (`isFormalWord`)
   - 이음동의어 (`synonyms`)
   - 금칙어 (`forbiddenWords`)
5. **저장** 버튼 클릭

### 단어 수정하기

1. 단어집 테이블에서 수정할 단어 클릭
2. **"수정"** 버튼 클릭
3. 정보 수정 후 **저장**

### 단어 삭제하기

1. 단어집 테이블에서 삭제할 단어 선택
2. **"삭제"** 버튼 클릭
3. 확인 대화상자에서 **확인**

### 파일 관리

#### 새 파일 생성

1. **"파일 관리"** 섹션에서 **"새 파일 생성"** 클릭
2. 파일명 입력 (예: `project-vocabulary.json`)
3. **생성** 버튼 클릭

#### 파일 이름 변경

1. **"파일 관리"** 섹션에서 파일 선택
2. **"이름 변경"** 버튼 클릭
3. 새 파일명 입력 후 **저장**

#### 파일 삭제

1. **"파일 관리"** 섹션에서 파일 선택
2. **"삭제"** 버튼 클릭
3. 확인 대화상자에서 **확인**

### 검색 및 필터링

- **검색**: 표준단어명, 영문약어, 영문명으로 검색
- **정렬**: 표준단어명, 영문약어, 생성일시 등으로 정렬
- **필터링**:
  - 중복 단어 필터
  - 도메인 미매핑 필터

### Excel 파일 업로드

1. **"파일 업로드"** 버튼 클릭
2. Excel 파일 선택 (`.xlsx` 또는 `.xls`)
3. 파일 형식 확인:
   - 필수 컬럼: 표준단어명, 영문약어, 영문명, 설명
   - 선택 컬럼: 도메인 분류명, 형식단어 여부 등
4. **업로드** 버튼 클릭
5. 업로드 결과 확인

### Excel 파일 다운로드

1. **"다운로드"** 버튼 클릭
2. Excel 파일로 저장

## 도메인 관리

### 도메인이란?

**도메인(Domain)**은 공통표준도메인 정보를 관리하는 표준 도메인 사전입니다.

**핵심 개념**:

- **도메인 그룹** (`domainGroup`): 공통표준도메인그룹명
- **도메인 분류** (`domainCategory`): 공통표준도메인분류명
- **표준 도메인명** (`standardDomainName`): 계산된 값 (예: "공통표준도메인그룹\_사용자분류")

### 도메인 추가하기

1. **도메인 페이지**로 이동 (`/domain/browse`)
2. **"새 도메인 추가"** 버튼 클릭
3. 필수 정보 입력:
   - 도메인 그룹명
   - 도메인 분류명
   - 물리 데이터 타입
4. 선택 정보 입력:
   - 데이터 길이
   - 형식 패턴
   - 설명
5. 하단 **변경 영향도** 카드에서 현재 참조 수 또는 즉시 영향 여부 확인
6. **저장** 버튼 클릭

### 도메인 수정/삭제

- 수정 모달 하단의 **변경 영향도** 카드에서 현재 참조 수를 먼저 확인합니다.
- 수정만 수행할 경우 참조 키와 자료형 스펙이 바뀌지 않으면 즉시 파급은 없습니다.
- 삭제 버튼을 누르면 확인창에 단어/용어/컬럼 참조 건수가 함께 표시됩니다.

### Excel 파일 업로드/다운로드

단어집과 동일한 방식으로 Excel 파일을 업로드/다운로드할 수 있습니다.

## 용어 생성 및 관리

### 용어란?

**용어(Term)**는 실제 데이터베이스에서 사용할 용어명, 컬럼명, 도메인명을 조합한 최종 용어입니다.

**핵심 개념**:

- **용어명** (`termName`): 한국어 용어 (예: "사용자\_이름")
- **컬럼명** (`columnName`): 영문 컬럼명 (예: "USER_NAME")
- **도메인명** (`domainName`): 표준 도메인명 (예: "공통표준도메인그룹\_사용자분류")

### 용어 추가하기

#### 방법 1: 수동 입력

1. **용어 페이지**로 이동 (`/term/browse`)
2. **"새 용어 추가"** 버튼 클릭
3. 필수 정보 입력:
   - 용어명 (한국어, 언더스코어로 구분)
   - 컬럼명 (영문, 언더스코어로 구분)
   - 도메인명
4. 하단 **변경 영향도** 카드에서 연결 컬럼 수와 도메인 존재 여부 확인
5. **저장** 버튼 클릭
6. 매핑 검증 결과 확인

#### 방법 2: 용어 자동 생성기 사용

[용어 자동 생성기 사용법](#용어-자동-생성기-사용법) 섹션 참조

### 용어 수정하기

1. 용어 테이블에서 수정할 용어 클릭
2. 입력값을 바꾸면 하단 **변경 영향도** 카드가 자동 갱신됩니다.
3. `현재 연결 컬럼`, `저장 후 연결 컬럼`, `끊기는 연결`, `표준화 영향 컬럼`을 확인합니다.
4. 정보 수정 후 **저장**
5. 매핑 검증 결과 확인

### 용어 삭제하기

1. 용어 테이블에서 삭제할 용어 선택
2. **"삭제"** 버튼 클릭
3. 확인 대화상자에서 **확인**

### 매핑 검증

용어는 다음 세 가지 매핑을 검증합니다:

1. **용어명 매핑** (`isMappedTerm`):
   - 용어명을 언더스코어로 분리한 각 단어가 단어집의 `standardName`에 존재해야 함
   - 예: "사용자\_이름" → ["사용자", "이름"] 각각이 단어집에 존재

2. **컬럼명 매핑** (`isMappedColumn`):
   - 컬럼명을 언더스코어로 분리한 각 단어가 단어집의 `abbreviation`에 존재해야 함
   - 예: "USER_NAME" → ["USER", "NAME"] 각각이 단어집에 존재

3. **도메인 매핑** (`isMappedDomain`):
   - 도메인명이 도메인의 `standardDomainName`과 정확히 일치해야 함

### 매핑 실패 시 자동 수정

매핑이 실패한 경우, 시스템이 자동으로 수정 제안을 제공합니다:

1. **단어집에 없는 단어 발견**:
   - 해당 단어를 단어집에 추가하는 제안
   - 유사한 단어 제안

2. **도메인명 불일치**:
   - 올바른 도메인명 제안
   - 도메인 자동 생성 제안

3. **자동 수정 적용**:
   - 제안된 수정 사항을 확인 후 **"자동 수정 적용"** 버튼 클릭

## 용어 자동 생성기 사용법

### 용어 자동 생성기란?

**용어 자동 생성기(Term Generator)**는 한국어 용어를 입력하면 단어집을 기반으로 가능한 영문 컬럼명 조합을 자동으로 생성하는 도구입니다. 변환 방향을 **영문→한글**로 바꾸면 영문 약어 조합을 연결된 단어집 기준의 한글 표준단어명으로 역조회할 수 있습니다.

### 사용 방법

1. **용어 페이지**로 이동 (`/term/browse`)
2. **"용어 변환기"** 섹션으로 스크롤
3. 변환 방향 버튼으로 방향 전환:
   - **한글→영문**: 한국어 용어로 영문 컬럼명 후보 생성 (기본값)
   - **영문→한글**: 영문 약어 조합으로 한글 표준단어명 조회
4. 한국어 용어 또는 영문 약어 입력:
   - 예: "사용자 이름"
   - 예: "VSTR_CNT_PRST_YM"
   - 여러 단어는 공백으로 구분
5. 생성된 조합 목록 확인:
   - 각 조합의 매핑 상태 표시
   - ✅: 매핑 성공
   - ❌: 매핑 실패 (원인 표시)
   - 영문→한글 모드는 조회 전용이므로 검증 상태와 용어 추가 버튼 없이 결과 복사만 제공합니다.
6. 한글→영문 모드에서는 적절한 조합 선택
7. 도메인명 선택 (자동 추천 가능)
8. **"용어 추가"** 버튼 클릭

### 생성 로직

1. 입력된 한국어 용어를 단어로 분리
2. 각 단어에 대해 단어집에서 가능한 영문약어 조회
3. 모든 조합 생성 (Cartesian Product)
4. 각 조합의 매핑 상태 검증
5. 결과 표시
6. 영문→한글 모드에서는 입력된 영문 약어를 분리해 단어집의 `abbreviation → standardName` 매핑으로 한글 표준단어명을 표시하며, 매칭되지 않는 약어는 `##`로 표시

### 예시

**입력**: "사용자 이름"

**단어집 데이터**:

- "사용자" → ["USER", "MEMBER"]
- "이름" → ["NAME", "FIRST_NAME"]

**생성된 조합**:

- `USER_NAME` ✅
- `USER_FIRST_NAME` ✅
- `MEMBER_NAME` ✅
- `MEMBER_FIRST_NAME` ✅

**영문→한글 조회 예시**

**입력**: `VSTR_CNT_PRST_YM`

**단어집 데이터**:

- `VSTR` → "방문자"
- `CNT` → "수"
- `PRST` → "현황"
- `YM` → "연월"

**조회 결과**:

- `방문자_수_현황_연월`

## 데이터베이스 설계 관리

이 기능은 데이터베이스의 논리적, 물리적 설계를 구성하는 요소들(데이터베이스, 엔터티, 속성, 테이블, 컬럼)을 관리합니다. 이 기능은 용어 표준화 모듈(단어집, 도메인, 용어)과는 **독립적으로 동작**합니다.

각 관리 페이지는 다음과 같은 공통적인 기능을 제공합니다.

- **조회**: 목록을 조회하고, 검색하며, 컬럼별로 필터링하고 정렬할 수 있습니다.
- **추가/수정/삭제**: 웹 인터페이스에서 직접 항목을 관리할 수 있습니다.
- **파일 관리**: 여러 개의 JSON 파일을 생성하고 관리할 수 있습니다.
- **업로드/다운로드**: Excel 파일을 이용해 대량으로 데이터를 등록하거나 내보낼 수 있습니다.
- **유효성 검사**: DB → 엔터티 → 속성 → 테이블 → 컬럼 관계와 표준 단어/용어/도메인 참조를 같은 패널에서 확인합니다.

### 데이터베이스(DB) 관리

- **경로**: `/database/browse`
- **설명**: 시스템에서 관리할 데이터베이스 인스턴스 정보를 등록하고 관리합니다.

### 엔터티(Entity) 관리

- **경로**: `/entity/browse`
- **설명**: 데이터베이스를 구성하는 논리적인 데이터 단위인 엔터티를 관리합니다.

### 속성(Attribute) 관리

- **경로**: `/attribute/browse`
- **설명**: 각 엔터티에 포함되는 속성들을 정의하고 관리합니다.

### 테이블(Table) 관리

- **경로**: `/table/browse`
- **설명**: 물리 데이터베이스에 생성되는 테이블의 명세를 관리합니다.

### 컬럼(Column) 관리

- **경로**: `/column/browse`
- **설명**: 각 테이블을 구성하는 컬럼들의 상세 정보를 관리합니다.
- 컬럼 추가/수정 팝업에서 `컬럼영문명`을 입력하면 하단 **표준 추천** 카드가 자동으로 갱신됩니다.
- 연결된 용어/도메인이 있으면 `컬럼한글명`, `도메인명`, `자료타입`, `자료길이`, `소수점길이`의 추천값을 바로 확인할 수 있습니다.
- `추천값 전체 적용` 또는 필드별 **적용** 버튼으로 현재 폼 값에 즉시 반영할 수 있습니다.

### 정의서 관계 유효성 검사

데이터베이스, 엔터티, 속성, 테이블, 컬럼 정의서 화면의 **유효성 검사** 버튼은 현재 선택 파일이 속한 8종 공통 파일 매핑(단어집/도메인/용어/DB/엔터티/속성/테이블/컬럼)을 기준으로 관계를 검증합니다.

- 검증 관계:
  - `DB(논리DB명)` ↔ `엔터티(논리DB명)`
  - `엔터티(schema, 엔터티명, 주식별자)` ↔ `속성(schema, 엔터티명, 속성명(PK))`
  - `엔터티(schema, 테이블한글명, 엔터티명)` ↔ `테이블(schema, 테이블한글명, 관련엔터티명)`
  - `테이블(주제영역, schema, 테이블영문명, 관련엔터티명)` ↔ `컬럼(주제영역, schema, 테이블영문명, 연관엔터티명)`
  - `속성(속성명, 필수입력여부, 참조 엔터티명+참조 속성명)` ↔ `컬럼(컬럼한글명, PK정보, FK정보)`
  - 테이블 한글/영문명은 단어집, 컬럼 한글/영문명은 용어집, 컬럼 도메인명은 용어의 도메인과 일치해야 합니다.
- 미매칭 항목은 후보 없음/단일 후보/복수 후보 상태로 표시됩니다.
- 미매칭 카드의 **문제 위치**는 정의서 종류별 최소 식별 키로 표시합니다. 예를 들어 속성은
  `schema + 엔터티명 + 속성명`, 컬럼은 `schema + 테이블영문명 + 컬럼영문명`으로 구분하며
  내부 ID는 표시하지 않습니다.
- 현재 정의서가 원인 정의서가 아니더라도 관계에 참여한 이슈는 현재 화면에 표시됩니다.
- **조치 대상 선택**에서 DB/엔터티/속성/테이블/컬럼/단어집/용어집/도메인 중
  수정할 정의서/항목을 고르면 조치 가이드와 실행 버튼이 해당 대상 기준으로 바뀝니다.
  기본 선택은 현재 정의서 고정이 아니라 누락 항목 추가 또는 안전한 기존 행 수정처럼 가장 직접적인 권장 조치 대상입니다.
- `자동 수정` 대상은 선택된 기존 행의 변경 필드만 적용합니다.
- `수동 수정` 또는 `신규 추가` 대상은 현재 화면 안에 해당 정의서 편집 팝업을 열어 직접 수정합니다.
  후보가 없거나 `autoFixable=false`인 항목도 이 수동 흐름만 사용할 수 있습니다.
- 관계 기대값/현재값 표시는 내부 매칭 키가 아니라 사용자 표시용 키이며, 복합 참조는
  `BKSP.사용자.사용자ID`처럼 `.` 구분자로 표시합니다.
- 표준 참조 검증에는 완전한 8종 공통 파일 매핑과 실제로 읽을 수 있는 단어집/도메인/용어 파일이 필요합니다.

## ERD 이미지 생성

ERD 화면(`/erd`)은 선택한 컬럼 정의서와 해당 컬럼 정의서에 매핑된 테이블 정의서를 기준으로 Graphviz SVG를 생성합니다. 화면의 관계선은 최초 표시부터 브라우저 오버레이로 통일해 테이블 배치 수정 전후의 표현 차이를 줄이며, 현재 세션 동안 조정한 테이블 위치를 SVG/PNG 다운로드에 반영할 수 있습니다. 이미지 품질은 `excel2image` 샘플 산출물의 정보 구성과 가독성을 참고하지만, 테이블 위치나 캔버스 크기의 픽셀 단위 동일성은 목표가 아닙니다.

### 사용 방법

1. **ERD 다이어그램** 화면으로 이동합니다.
2. 좌측 제어판의 **정의서 파일** 카드에서 컬럼 정의서를 선택합니다.
   - 테이블 정의서는 컬럼 정의서의 기존 매핑을 따라 자동 선택됩니다.
   - 설정 버튼에서 컬럼 정의서의 매핑 상태를 확인하거나 수정할 수 있습니다.
3. 본문 상단의 **조회 조건**과 **테이블 선택** 카드에서 필요한 조건을 조정합니다.
   - **표시 모드**: 논리 이미지 또는 물리 이미지
   - **주제영역**: 사용 가능한 주제영역 중 하나가 자동 선택되며 `전체` 옵션은 제공하지 않습니다.
   - **스키마**: 선택된 주제영역에서 사용 가능한 스키마 중 하나가 자동 선택되며 `전체` 옵션은 제공하지 않습니다.
   - **사업범위여부**: 전체, 사업범위, 사업범위 외
   - **FK 외부참조 포함**: 필터 밖 참조 테이블을 외부 노드로 함께 표시할지 선택
   - **테이블 선택**: 최초에는 현재 조건 결과의 전체 테이블이 선택됩니다. 기본 상태는 접혀 있으며, **수정** 버튼을 누르면 테이블명 검색(영문/한글/스키마/주제영역)과 다중 선택을 조정할 수 있습니다.
4. 조건 변경은 별도 적용 버튼 없이 즉시 SVG 미리보기와 현재 배치 다운로드 기준에 반영됩니다.
5. 미리보기 상단 보기 도구로 다이어그램을 탐색합니다.
   - **맞춤**: 현재 ERD를 미리보기 영역 안에 맞추고 가운데 정렬합니다.
   - **100%**: 원본 배율로 전환합니다.
   - **축소/확대**: 25%~400% 범위에서 배율을 조정합니다.
   - **초기화**: 현재 조건의 ERD를 다시 맞춤 상태로 되돌립니다.
   - **배치 수정**: 현재 화면에서 테이블을 드래그해 위치를 조정합니다.
   - **배치 초기화**: 드래그로 바꾼 테이블 위치와 수동 관계선을 원래 렌더링 상태로 되돌립니다.
   - 배치 수정 모드가 아닐 때 다이어그램 영역을 마우스로 드래그하면 큰 ERD를 이동하며 확인할 수 있습니다.
6. 필요하면 **테이블 선택** 카드 오른쪽의 **이미지 다운로드** 카드에서 **SVG 다운로드** 또는
   **PNG 다운로드**를 클릭합니다. 배치 수정으로 조정한 테이블 위치와 관계선은 현재 화면의
   SVG/PNG에 반영됩니다.
7. **데이터 연관관계 요약**을 펼치면 ERD 조회 기준의 정의서 관계 미매칭, 오류, 경고
   건수를 확인할 수 있습니다. ERD는 조회 전용이며, 상세 조치 가이드와 실제 수정은 각
   정의서 화면의 유효성 검사 패널과 편집 팝업에서 실행합니다.

### 데이터 기준

- 테이블명/주제영역/스키마는 선택한 컬럼 정의서에 매핑된 테이블 정의서 기준입니다.
- 논리 이미지는 테이블/컬럼 한글명만, 물리 이미지는 스키마 접두어를 제외한 테이블 영문명과 컬럼 영문명만 이름으로 표시합니다. 반대쪽 명칭은 보조명으로 섞어 표시하지 않습니다.
- 컬럼 목록, PK/FK/NN, 데이터 타입, 자료 길이, 소수점 길이는 컬럼 정의서 기준입니다.
- 이미지 내부에서는 별도 그래프 제목, 주제영역 행, 사업범위 문구 행, `PK/FK/컬럼/타입/NN` 머리행, 관계선 라벨을 표시하지 않습니다.
- 사업범위 테이블 헤더는 `#4A90E2`, 사업범위 외 테이블 헤더는 `#9B9B9B` 계열로 색상만 구분하고, 외부 참조 테이블/관계는 회색/점선 스타일로 표시합니다.
- FK 관계선은 `FK정보`에 명시된 참조만 사용합니다. 정식 형식은 `schema.table.column`이며, `table.column`/`table:column`은 같은 스키마 안의 명시적 축약형으로만 해석합니다. `Y`/`YES`는 FK 표시만 남기고 관계선을 만들지 않으며, `PK01`처럼 PK 순번이 FK 칸에 들어온 값은 FK로 해석하지 않습니다.
- 같은 두 테이블 사이에 여러 FK가 있어도 관계선은 하나로 축약하며, 각 컬럼 행의 FK 표시로 컬럼 단위 정보를 확인합니다.
- 컬럼 정의서에는 테이블 논리명이 없으므로 논리 ERD의 테이블명은 테이블 정의서의 `테이블한글명`을 사용합니다.
- 화면 하단의 관계 수는 Graphviz 이미지 관계 기준입니다. 기존 JSON ERDData의 구조적 `edges` 수와 혼동되지 않도록 `관계`로 표시합니다.
- 관계가 없거나 희박한 테이블 조합은 단일 세로 스택이 아니라 격자형에 가까운 packing을 적용해 긴 빈 여백을 줄입니다. 화면 미리보기와 SVG/PNG 다운로드는 같은 배치 정책을 사용합니다.
- 화면에서 수정한 테이블 배치는 브라우저 현재 세션의 미리보기 상태에만 적용됩니다. 파일, 조건, 선택 테이블, 새로고침이 바뀌면 수동 배치는 초기화됩니다.
- 관계선은 기존 Graphviz 방향성을 유지하되 최초 표시부터 브라우저 오버레이로 다시 그리며, crow-foot/tee 기호가 테이블 경계를 침범하지 않도록 경계 밖 여백을 둡니다.
- PNG 다운로드는 현재 SVG를 브라우저에서 PNG로 변환합니다. 배치가 너무 커 브라우저 변환 한도를 넘으면 SVG로 다운로드하거나 배치를 줄여 다시 시도합니다. 서버 렌더 API의 PNG는 기본 192DPI로 생성되며, 더 높은 화질이 필요하면 `pngDpi` 값을 높일 수 있습니다. 무손실 확대/인쇄에는 SVG 다운로드가 가장 선명합니다.

### Graphviz 설치

로컬 실행 시 서버에서 `dot` CLI를 실행할 수 있어야 합니다. Graphviz가 없으면 ERD 이미지 영역 또는 API 오류 응답에 설치 안내가 표시됩니다. Docker 이미지는 Graphviz와 CJK 폰트를 포함합니다.

## 데이터 소스 관리

### 데이터 소스란?

**데이터 소스(Data Source)**는 실제 DB 접속 정보를 저장하고, 이후 데이터 프로파일링이나 품질 검사를 연결할 기반이 되는 내부용 연결 정의입니다.

**현재 지원 범위**:

- 지원 DBMS: **PostgreSQL**
- 경로: `/data-source/browse`
- 저장 위치: `static/data/settings/data-sources.json`

### 데이터 소스 추가하기

1. **데이터 소스 페이지**로 이동 (`/data-source/browse`)
2. **"새 데이터 소스 추가"** 버튼 클릭
3. 연결 정보 입력:
   - 연결 이름
   - 호스트 / 포트
   - 데이터베이스 / 스키마
   - 사용자명 / 비밀번호
   - SSL 사용 여부
   - 연결 타임아웃(초)
4. **"연결 테스트 실행"**으로 저장 전 접속 확인
5. **"저장"** 버튼 클릭

### 데이터 소스 수정하기

1. 목록에서 수정할 연결의 **"수정"** 버튼 클릭
2. 필요한 값을 변경
3. 비밀번호를 바꾸지 않을 경우 **비워 둡니다**
4. 필요하면 다시 **연결 테스트 실행**
5. **"저장"** 버튼 클릭

### 데이터 소스 삭제하기

1. 목록에서 삭제할 연결의 **"삭제"** 버튼 클릭
2. 확인 대화상자에서 삭제 확정

> 삭제는 저장된 접속 정보만 제거하며 실제 PostgreSQL DB에는 영향을 주지 않습니다.

### 저장된 연결 테스트하기

- 목록의 **"연결 테스트"** 버튼으로 저장된 연결 정의를 바로 검사할 수 있습니다.
- 가장 최근 실행 결과는 화면 상단 카드에 표시됩니다.
- 성공 시 서버 버전, 대상 DB, 스키마, 지연시간을 확인할 수 있습니다.

### 검색하기

- 연결 이름
- 호스트
- 데이터베이스
- 사용자명

위 4개 기준으로 부분 일치 또는 정확 일치 검색을 사용할 수 있습니다.

## 품질 규칙 관리

### 품질 규칙이란?

**품질 규칙(Quality Rule)**은 PostgreSQL 프로파일링 결과에 바로 적용할 최소 품질 기준입니다.  
규칙을 저장해 두면 프로파일링 실행 시 같은 응답 안에서 즉시 평가됩니다.

**현재 지원 범위**:

- 경로: `/quality-rule/browse`
- 저장 위치: `static/data/settings/quality-rules.json`
- table 범위 메트릭: `rowCount`
- column 범위 메트릭: `nullCount`, `nullRatio`, `distinctCount`, `distinctRatio`, `minLength`, `maxLength`
- 연산자: `gte`, `lte`, `eq`

### 품질 규칙 추가하기

1. **품질 규칙 페이지**로 이동 (`/quality-rule/browse`)
2. **"새 품질 규칙 추가"** 버튼 클릭
3. 규칙 정보 입력:
   - 규칙 이름
   - 심각도 (`error`, `warning`, `info`)
   - 범위 (`table`, `column`)
   - 메트릭 / 연산자 / 기준값
   - 스키마 / 테이블 / 컬럼 패턴
4. 필요하면 **활성화** 상태를 조정
5. **"저장"** 버튼 클릭

### 타깃 패턴 지정

- `schemaPattern`, `tablePattern`, `columnPattern`은 `*` 와일드카드를 지원합니다.
- 예시:
  - `public` : 정확히 `public` 스키마만 적용
  - `tb_*` : `tb_`로 시작하는 모든 테이블에 적용
  - `*_id` : `_id`로 끝나는 모든 컬럼에 적용
- 값을 비워두면 해당 축 전체에 적용됩니다.

### 규칙 수정/삭제

1. 목록의 **"수정"** 버튼으로 기존 기준을 바로 바꿀 수 있습니다.
2. **"삭제"** 버튼을 누르면 프로파일링 평가 대상에서 즉시 제외됩니다.

### 규칙 설계 팁

- table 범위는 빈 테이블 탐지나 최소 적재 건수 체크에 적합합니다.
- column 범위는 NULL 허용 수준, 중복 가능성, 문자열 길이 상한/하한 점검에 적합합니다.
- 초기에는 `warning` 위주로 시작하고, 안정화된 기준만 `error`로 올리는 편이 운영에 유리합니다.

## 데이터 프로파일링

### 프로파일링이란?

**데이터 프로파일링(Data Profiling)**은 저장된 PostgreSQL 연결을 이용해 실제 테이블의 분포와 품질 힌트를 즉시 확인하는 기능입니다.

**현재 지원 범위**:

- 경로: `/profiling/browse`
- 대상: 저장된 PostgreSQL 데이터 소스의 사용자 테이블
- 결과: `행 수`, `NULL 건수/비율`, `distinct 건수/비율`, `최소/최대 길이`
- 규칙 평가: 저장된 활성 품질 규칙이 있으면 `품질 규칙 평가` 요약과 위반 목록을 함께 표시
- 이력 저장: 없음 (실행 결과는 화면에서만 확인)

### 프로파일링 실행하기

1. **프로파일링 페이지**로 이동 (`/profiling/browse`)
2. **저장된 데이터 소스**를 선택
3. 상단 **대상 선택 카드 안의 "테이블 불러오기"** 버튼 클릭
4. 조회된 목록에서 원하는 테이블의 **"프로파일링 실행"** 버튼 클릭
5. 실행이 끝나면 화면이 **상단 결과 카드**로 바로 이동합니다.
6. 결과 카드에서 컬럼별 지표를 확인
7. 저장된 품질 규칙이 있다면 같은 화면에서 위반 요약과 세부 목록까지 확인

### 대상 테이블 찾기

- 스키마 필터로 원하는 스키마만 볼 수 있습니다.
- 검색창에서 `schema.table` 또는 테이블명 일부로 대상 테이블을 좁힐 수 있습니다.
- 테이블 목록에는 예상 행 수와 컬럼 수도 함께 표시됩니다.
- 대상 목록은 **페이지당 10건**씩 표시되며, 하단 페이지네이션으로 다음 목록을 확인할 수 있습니다.

### 결과 해석

- **행 수**: 실제 `COUNT(*)` 기준의 총 레코드 수
- **NULL 비율**: 전체 행 대비 NULL 값 비중
- **Distinct 비율**: 전체 행 대비 고유값 비중
- **길이 범위**: 값의 텍스트 길이 기준 최소/최대
- **품질 규칙 평가**: 저장된 기준과 실제값을 비교한 통과/실패 요약
- **위반 목록**: 규칙명, 대상 컬럼, 실제값/기준값, 심각도 확인

> 대용량 테이블은 정확한 건수를 계산하므로 실행 시간이 길어질 수 있습니다.

## 스냅샷 관리

### 스냅샷이란?

**스냅샷(Snapshot)**은 현재 선택된 8종 파일 번들의 상태를 통째로 저장해 두었다가, 필요할 때 같은 번들 상태로 빠르게 되돌리는 복구 기능입니다.

**현재 지원 범위**:

- 경로: `/snapshot/browse`
- 저장 위치: `static/data/settings/design-snapshots.json`
- 대상: `vocabulary`, `domain`, `term`, `database`, `entity`, `attribute`, `table`, `column`
- 복원 시 공통 파일 매핑 번들도 함께 재적용

### 스냅샷 저장하기

1. **스냅샷 페이지**로 이동 (`/snapshot/browse`)
2. 우상단의 **"스냅샷 추가"** 버튼 클릭
3. 팝업에서 **대상 번들** 표시명을 선택
   - 예: `bksp 번들`, `ecobank + 기본 DB설계`
   - 아래 **포함되는 파일** 영역에서 실제 8종 파일 구성을 바로 확인
4. 필요하면 **스냅샷명**, **설명** 입력
5. **"스냅샷 저장"** 버튼 클릭
6. 팝업이 닫히고 목록에 새 복구 지점이 추가된 것을 확인

### 스냅샷 복원하기

1. 목록에서 되돌릴 스냅샷의 **"복원"** 버튼 클릭
2. 확인 대화상자에서 현재 번들을 덮어쓸 것인지 확인
3. 복원 완료 후 **최근 복원 시각**과 성공 토스트를 확인

> 복원은 실제 JSON 데이터를 다시 저장하는 작업입니다. 복원 이후 상태가 기준이 되므로, 큰 변경 전에는 새 스냅샷을 먼저 남기는 편이 안전합니다.

### 스냅샷 삭제하기

1. 목록에서 삭제할 스냅샷의 **"삭제"** 버튼 클릭
2. 확인 대화상자에서 삭제 확정

> 스냅샷 삭제는 복구 포인트 정보만 제거하며 실제 단어집/도메인/용어/정의서 데이터 파일은 유지됩니다.

### 스냅샷 찾기

- 스냅샷명
- 컬럼 파일명
- 용어 파일명

기준으로 검색할 수 있습니다. 정확 일치 또는 부분 일치 검색을 상황에 따라 선택합니다.

## 파일 업로드 가이드

### 업로드 모드

파일 업로드는 **단순 교체** 한 가지 흐름으로 동작합니다.

#### 단순 교체 흐름

- 기존 데이터를 완전히 삭제하고 새로운 데이터로 교체
- 업로드 직전 JSON 본문은 파일별 교체 이력으로 저장
- 업로드 후 현재 파일 매핑을 자동 저장
- 같은 파일 관리 팝업에서 최근 교체 이력을 조회하거나 복원 가능

> ⚠️ **참고**: 검증 교체, 병합, 덮어쓰기 모드는 더 이상 별도 선택지로 제공되지 않습니다.

### Excel 파일 형식

#### 단어집 업로드 형식

| 표준단어명 | 영문약어 | 영문명 | 설명          | 도메인 분류명 | 형식단어 여부 |
| ---------- | -------- | ------ | ------------- | ------------- | ------------- |
| 사용자     | USER     | User   | 시스템 사용자 | 사용자분류    | Y             |
| 이름       | NAME     | Name   | 이름 정보     | 사용자분류    | Y             |

**필수 컬럼**: 표준단어명, 영문약어, 영문명, 설명

**선택 컬럼**: 도메인 분류명, 형식단어 여부, 이음동의어, 금칙어

#### 도메인 업로드 형식

| 도메인 그룹명      | 도메인 분류명 | 물리 데이터 타입 | 데이터 길이 | 형식 패턴 | 설명             |
| ------------------ | ------------- | ---------------- | ----------- | --------- | ---------------- |
| 공통표준도메인그룹 | 사용자분류    | VARCHAR          | 50          | -         | 사용자 분류 정보 |

**필수 컬럼**: 도메인 그룹명, 도메인 분류명, 물리 데이터 타입

**선택 컬럼**: 데이터 길이, 형식 패턴, 설명

#### 용어 업로드 형식

| 용어명       | 컬럼명    | 도메인명                       |
| ------------ | --------- | ------------------------------ |
| 사용자\_이름 | USER_NAME | 공통표준도메인그룹\_사용자분류 |

**필수 컬럼**: 용어명, 컬럼명, 도메인명

#### 데이터베이스 정의서 업로드 형식

| 기관명 | 부서명 | 적용업무 | 관련법령       | 논리DB명 | 물리DB명 | 구축일자   | DB설명         | DBMS정보   | 운영체제정보 | 수집제외사유 |
| ------ | ------ | -------- | -------------- | -------- | -------- | ---------- | -------------- | ---------- | ------------ | ------------ |
| 기관A  | 개발부 | 고객관리 | 개인정보보호법 | 고객DB   | CUST_DB  | 2024-01-01 | 고객 정보 관리 | Oracle 19c | Linux        | -            |

**필수 컬럼**: 기관명, 부서명, 적용업무

`번호` 또는 `순번` 열이 있어도 헤더 기준으로 매핑하므로 한 칸 밀림 없이 업로드됩니다.

#### 엔터티 정의서 업로드 형식

| 논리DB명 | 스키마명 | 엔터티명 | 엔터티설명     | 주식별자 | 수퍼타입엔터티명 | 테이블한글명 |
| -------- | -------- | -------- | -------------- | -------- | ---------------- | ------------ |
| 고객DB   | CUST     | 고객     | 고객 기본 정보 | 고객ID   | -                | 고객정보     |

**필수 컬럼**: 논리DB명, 스키마명, 엔터티명

#### 속성 정의서 업로드 형식

| 스키마명 | 엔터티명 | 속성명 | 속성유형 | 필수입력여부 | 식별자여부 | 참조엔터티명 |
| -------- | -------- | ------ | -------- | ------------ | ---------- | ------------ |
| CUST     | 고객     | 고객ID | VARCHAR  | Y            | Y          | -            |

**필수 컬럼**: 스키마명, 엔터티명, 속성명

#### 테이블 정의서 업로드 형식

| 순번 | 물리DB명 | 테이블소유자 | 주제영역 | schema | 테이블 영문명 | 테이블한글명 | 테이블 유형 | 공개/비공개 여부 |
| ---- | -------- | ------------ | -------- | ------ | ------------- | ------------ | ----------- | ---------------- |
| 1    | CUST_DB  | DBA          | 고객     | CUST   | TB_CUSTOMER   | 고객테이블   | TABLE       | 공개             |

**필수 컬럼**: 스키마명/schema, 테이블 영문명/테이블영문명, 테이블한글명. `schema` 또는 `스키마명` 헤더를 모두 지원합니다.
`순번` 또는 `번호` 열이 있어도 헤더 기준으로 매핑하므로 한 칸 밀림 없이 업로드됩니다.

#### 컬럼 정의서 업로드 형식

| 사업범위여부 | 주제영역 | 스키마명 | 테이블영문명 | 컬럼영문명 | 컬럼한글명 | 자료타입 | 자료길이 | ... |
| ------------ | -------- | -------- | ------------ | ---------- | ---------- | -------- | -------- | --- |
| Y            | 고객     | CUST     | TB_CUSTOMER  | CUST_ID    | 고객ID     | VARCHAR  | 50       | ... |

**필수 컬럼**: 스키마명/schema, 컬럼영문명, 자료길이, PK정보. `컬럼` 시트가 있으면 백업 시트보다 우선 사용되며, `자료 타입`처럼 공백/개행이 포함된 헤더도 지원합니다.

### 업로드 절차

1. **Excel 파일 준비**
   - 위 형식에 맞게 데이터 작성
   - 첫 번째 행은 헤더 (컬럼명)

2. **파일 관리 열기**
   - 해당 페이지에서 **"파일 관리"** 버튼 클릭
   - **"파일 업로드"** 탭 선택

3. **대상 파일 선택**
   - 업로드할 대상 파일 선택 (드롭다운)

4. **파일 업로드**
   - 드래그앤드롭 또는 클릭하여 파일 선택
   - **업로드** 버튼 클릭
   - 확인 대화상자에서 **확인**

5. **업로드 결과 확인**
   - 성공한 항목 수
   - 실패한 항목 및 원인
   - 교체 이력 저장 및 파일 매핑 저장 결과

### 파일 관리 팝업 기능

파일 관리 팝업에서는 다음 기능을 사용할 수 있습니다:

1. **파일 목록 탭**
   - **새 파일 생성**: 파일명 입력 후 버튼 클릭 또는 **Enter 키**로 생성
   - **파일 이름 변경**: 파일 옆 ✏️ 아이콘 클릭
   - **파일 삭제**: 파일 옆 🗑️ 아이콘 클릭
   - **시스템 파일 표시** 토글: 시스템 파일(예: domain.json) 표시/숨김
     - ⚠️ 이 설정은 해당 목록 페이지(browse 페이지)에도 영향을 줍니다

2. **파일 매핑 탭**
   - **파일 매핑 설정**: 현재 파일과 연결할 다른 파일을 지정
   - `표준 용어`: 단어집, 도메인, 용어 파일
   - `DB 설계`: 데이터베이스, 엔터티, 속성, 테이블, 컬럼 정의서 파일
   - 현재 열어 둔 파일 유형은 제외되고, 나머지 연결 대상만 그룹별로 표시됩니다
   - 저장이 필요한 동기화 기능은 이 탭에서 함께 실행합니다

3. **파일 업로드 탭**
   - 대상 파일 선택
   - 단순 교체 흐름으로 파일 업로드 (드래그앤드롭 지원)
   - 업로드 교체 이력 조회 및 복원

### 정의서 추가/수정 팝업

정의서(데이터베이스, 엔터티, 속성, 테이블, 컬럼) 추가/수정 시 팝업 편집기가 표시됩니다:

- 모든 입력 필드는 **1행에 1개씩** 배치되어 가독성이 높습니다
- 필수 필드는 별도 표시됩니다
- 컬럼 팝업은 `컬럼영문명` 기준으로 **표준 추천** 카드를 자동 계산합니다
- 추천 카드에서 매핑된 용어/도메인, 경고, 필드별 변경 후보를 보고 바로 적용할 수 있습니다
- 저장 버튼을 클릭하여 변경사항을 저장합니다

## 매핑 동기화 가이드

### Vocabulary ↔ Domain 동기화

단어집과 도메인을 `domainCategory`를 기준으로 자동 매핑합니다.

#### 동기화 방법

1. **단어집 페이지**로 이동 (`/browse`)
2. **"도메인 동기화"** 버튼 클릭
3. 동기화 결과 확인:
   - 매핑 성공한 단어 수
   - 매핑 실패한 단어 및 원인

#### 동기화 로직

- `VocabularyEntry.domainCategory` ↔ `DomainEntry.domainCategory` 매칭
- 매핑 성공 시 `VocabularyEntry.domainGroup`에 `DomainEntry.domainGroup` 자동 설정
- `VocabularyEntry.isDomainCategoryMapped` 플래그로 매핑 상태 표시

### Term 매핑 동기화

용어의 매핑 상태를 재검증합니다.

#### 동기화 방법

1. **용어 페이지**로 이동 (`/term/browse`)
2. **"매핑 동기화"** 버튼 클릭
3. 동기화 결과 확인:
   - 매핑 성공한 용어 수
   - 매핑 실패한 용어 및 원인
   - 자동 수정 제안

#### 동기화 로직

1. 모든 용어의 매핑 상태 재검증
2. 단어집/도메인 변경사항 반영
3. 매핑 실패 시 자동 수정 제안

## 문제 해결 (FAQ)

### Q1. 용어 생성 시 매핑이 실패합니다

**원인**:

- 단어집에 해당 단어가 없음
- 도메인명이 정확하지 않음

**해결 방법**:

1. 단어집에 누락된 단어 추가
2. 도메인명 확인 및 수정
3. 자동 수정 제안 활용

### Q2. Excel 파일 업로드가 실패합니다

**원인**:

- 파일 형식이 올바르지 않음
- 필수 컬럼 누락
- 데이터 형식 오류

**해결 방법**:

1. 파일 형식 확인 (헤더 행 포함)
2. 필수 컬럼 확인
3. 데이터 형식 확인 (날짜, 숫자 등)
4. 오류 메시지 확인 후 수정

### Q3. 도메인 동기화가 되지 않습니다

**원인**:

- `domainCategory`가 일치하지 않음
- 도메인 데이터가 없음

**해결 방법**:

1. 단어집의 `domainCategory` 확인
2. 도메인의 `domainCategory` 확인
3. 일치하지 않으면 수정

### Q4. 용어 자동 생성기가 조합을 생성하지 않습니다

**원인**:

- 단어집에 해당 단어가 없음
- 입력 형식이 올바르지 않음

**해결 방법**:

1. 단어집에 해당 단어 추가
2. 입력 형식 확인 (공백으로 단어 구분)

### Q5. 파일이 삭제되지 않습니다

**원인**:

- 파일이 사용 중
- 권한 문제

**해결 방법**:

1. 다른 페이지에서 파일 사용 중인지 확인
2. 브라우저 새로고침 후 재시도
3. 서버 재시작
